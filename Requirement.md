# 要件定義書 — 案件管理Webアプリケーション

**作成日**: 2026-03-26
**最終更新**: 2026-03-26
**ステータス**: 確定

---

## 1. アプリケーション概要

案件・受注情報を一元管理するWebアプリケーション。
バックエンドデータストアにNotionDBを使用し、ユーザー認証・アカウント管理にSupabaseを使用する。

---

## 2. 使用技術・外部サービス

| 種別 | 内容 |
|---|---|
| ユーザー認証・管理 | Supabase (Auth + Storage) |
| データベース | Notion API (案件一覧DB、受注一覧DB) |
| アイコン画像保存 | Supabase Storage |

> フロントエンドフレームワーク・ホスティング先は設計フェーズで決定する。

## 4. 機能要件

### 4.1 認証

| 項目 | 内容 |
|---|---|
| ログイン方式 | メール+パスワード / Googleログイン（OAuth） |
| 認証基盤 | Supabase Auth |
| 未ログイン時 | `/login` にリダイレクト |
| ユーザー登録 | **管理者招待制**。管理者がメールアドレスを入力して招待メールを送信。受信者が招待リンクから登録 |
| ログアウト | ヘッダーのアカウントメニューから実行。ログアウト後は `/login` へリダイレクト |

---

### 4.2 ユーザー権限

システム内のユーザーは以下2種類の権限ロールを持つ。

| ロール | 説明 |
|---|---|
| `admin`（管理者） | 全機能を利用可能。ユーザー管理・招待・Notion連携・AI設定・Notion Token設定が可能 |
| `member`（一般ユーザー） | 案件・受注の閲覧・登録・編集・詳細表示が可能。プロフィール編集が可能 |

- 初期ユーザー（最初に登録したユーザー）は自動的に `admin` になる
- 管理者は他ユーザーのロールを `admin` / `member` に変更可能

#### 操作権限マトリクス

| 機能 | admin | member |
|---|---|---|
| 案件・受注の閲覧 | ○ | ○ |
| 案件・受注の新規作成 | ○ | ○ |
| 案件・受注の編集 | ○ | ○ |
| 案件・受注の削除 | ✕ | ✕ |
| ユーザー招待・権限変更 | ○ | ✕ |
| Notion連携設定 | ○ | ✕ |
| Notion Integration Token設定 | ○ | ✕ |
| AI設定（プロバイダー・APIキー） | ○ | ✕ |

### 4.5 メイン画面 — 案件一覧（`/projects`）

#### カード一覧表示
- NotionDB **案件一覧DB** のレコードをカード形式で一覧表示
- カード内表示項目:
  - 案件名
  - 進捗状況
  - 種別
  - プロジェクト期限
- カード内に **「詳細」ボタン** → 押下で `/projects/[id]` へ遷移

#### 検索・フィルタ

| 機能 | 内容 |
|---|---|
| キーワード検索 | 案件名での部分一致検索 |
| 進捗状況フィルタ | 見込み / 進行中 / 契約 / 運用中 / 納品済み で絞り込み |
| 種別フィルタ | 個人 / 企業 で絞り込み |
| 業界フィルタ | マルチセレクトの選択値で絞り込み |

#### ページネーション
- 1ページあたりの表示件数: 20件
- Notion APIの100件上限（`page_size`）に対応したカーソルページネーション

#### 新規作成
- 画面上部に **「新規作成」ボタン** を設置
- ボタン押下でフォームをモーダル表示
- フォーム送信でNotionDBに新規レコードを登録

**新規作成フォーム入力項目:**

| フィールド名 | 型 | 必須 | 備考 |
|---|---|---|---|
| 案件名 | テキスト | ◎ | |
| 種別 | セレクト | | 個人 / 企業 |
| 業界 | マルチセレクト | | |
| 進捗状況 | セレクト | | 見込み / 進行中 / 契約 / 運用中 / 納品済み |
| プロジェクト期限 | 日付 | | |
| 初期費用 | 数値 | | 円単位 |
| 月額費用 | 数値 | | 円単位 |
| 契約内容 | テキストエリア | | |
| 特記 | テキストエリア | | |

※ `受注費総計`・`案件進捗` はNotionフォーミュラのため入力不要

### 4.9 設定画面（`/settings`）

ヘッダーのアカウントメニューから遷移。
管理者と一般ユーザーで表示できるセクションを分ける。

---

#### 共通設定（全ユーザー表示）

| 設定項目 | 内容 |
|---|---|
| アカウント名 | 表示名の変更 |
| アイコン | 写真のアップロード・変更（JPEG/PNG、最大5MB）。Supabase Storageに保存 |

---

#### 管理者のみ表示されるセクション

**ユーザー管理**

| 設定項目 | 内容 |
|---|---|
| ユーザー招待 | メールアドレスを入力して招待メールを送信（Supabase Auth の招待機能を使用） |
| ユーザー一覧 | 登録済みユーザーの一覧表示（名前・メール・権限ロール） |
| 権限変更 | 各ユーザーのロールを `admin` / `member` に変更 |

**Notion連携**

| 設定項目 | 内容 |
|---|---|
| Integration Token | Notion Integration Token を入力・保存（表示はマスキング）。暗号化してSupabaseに保存 |
| ユーザー紐づけ | SupabaseユーザーとNotionユーザーIDの紐づけ設定。Notionユーザー一覧を取得して対応するSupabaseユーザーに割り当てる |

**AI設定**

| 設定項目 | 内容 |
|---|---|
| 使用プロバイダー選択 | OpenAI / Anthropic (Claude) / Google (Gemini) の中から使用する1つを選択・切り替え |
| APIキー（OpenAI） | OpenAIのAPIキーを入力・保存（表示はマスキング） |
| APIキー（Anthropic） | AnthropicのAPIキーを入力・保存（表示はマスキング） |
| APIキー（Google） | Google AIのAPIキーを入力・保存（表示はマスキング） |

> 各APIキーは登録可能。「使用プロバイダー」で選択されたもののみがAI機能で使われる。
> APIキーはSupabase上に暗号化して保存し、フロントエンドには平文で渡さない。

## 6. Supabase 構成

### 6.1 認証
- Supabase Auth によるユーザー管理
- ログイン方式: メール+パスワード / Googleログイン（OAuth）
- 新規登録: 管理者招待制（Supabase Auth の `inviteUserByEmail` を使用）

### 6.2 Storage
- バケット名: `avatars`
- アクセス権: 認証済みユーザーのみ（自分のファイルのみ操作可）
- 最大ファイルサイズ: 5MB
- 対応形式: JPEG / PNG

### 6.3 テーブル構成

#### `profiles` テーブル（ユーザー付加情報）

| カラム名 | 型 | 備考 |
|---|---|---|
| id | uuid | Supabase Auth の `user.id` と1対1紐づけ（PK） |
| display_name | text | 表示名 |
| avatar_url | text | Supabase Storage の URL |
| role | text | `admin` / `member`（デフォルト: `member`） |
| notion_user_id | text | NotionユーザーID（管理者が紐づけ設定） |
| created_at | timestamptz | |
| updated_at | timestamptz | |

#### `system_settings` テーブル（システム全体設定・シングルトン）

| カラム名 | 型 | 備考 |
|---|---|---|
| id | uuid | PK（レコードは常に1件） |
| notion_token_encrypted | text | Notion Integration Token（暗号化済み） |
| active_ai_provider | text | 現在使用中のAIプロバイダー（`openai` / `anthropic` / `google`） |
| created_at | timestamptz | |
| updated_at | timestamptz | |

#### `ai_api_keys` テーブル（AIプロバイダーごとのAPIキー）

| カラム名 | 型 | 備考 |
|---|---|---|
| id | uuid | PK |
| provider | text | `openai` / `anthropic` / `google`（1プロバイダー1レコード） |
| api_key_encrypted | text | 暗号化済みAPIキー |
| created_at | timestamptz | |
| updated_at | timestamptz | |

### 6.4 Row Level Security（RLS）方針

| テーブル | SELECT | INSERT / UPDATE | DELETE |
|---|---|---|---|
| `profiles`（自分のレコード） | ○ 認証ユーザー | ○ 本人のみ | ✕ |
| `profiles`（role列の変更） | — | ○ admin のみ | — |
| `profiles`（全件取得） | ○ admin のみ | — | — |
| `system_settings` | ○ admin のみ | ○ admin のみ | ✕ |
| `ai_api_keys` | ○ admin のみ | ○ admin のみ | ○ admin のみ |

## 7. 非機能要件

| 項目 | 内容 |
|---|---|
| 認証 | ログイン済みユーザーのみアクセス可能 |
| レスポンシブ | PC・モバイル両対応 |
| Notion APIレート制限 | 1秒あたり3リクエスト制限への対応（エクスポネンシャルバックオフによるリトライ） |
| 機密情報保管 | Notion Token・AIのAPIキーは暗号化してSupabaseに保存。フロントエンドには平文で渡さない |
| ページネーション | Notion APIカーソルに対応。1ページ20件表示 |

## 8. 対象外（今回スコープ外）

- モバイルアプリ（ネイティブ）
- 添付ファイルのアップロード（Notionで直接管理・詳細画面での表示のみ）
